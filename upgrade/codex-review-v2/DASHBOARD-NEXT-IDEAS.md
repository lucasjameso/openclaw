# Dashboard Next Ideas

Shortlist of the next upgrades that could make Forge feel even more like a living operating system.

## Immediate

- Strengthen the sticky topbar so content scrolls under a darker glass backplate on desktop.
- Add a small `content wave` module to Command or Broadcast showing the top 3 hooks ready to publish next.
- Add a `share this snapshot` affordance to Story so the page behaves more like a landing page than an internal dashboard.
- Add an explicit `follow @Forge_Builds` card to Broadcast near the hero so a visitor always has a next action.

## Short Sprint

- Build a `Launches This Week` rail that combines posted tweets, review-ready visuals, and upcoming product drops.
- Add `receipts mode` to Story with expandable proof points: spend, task counts, post counts, and major infra changes.
- Add a `human bottleneck heatmap` to Review so Lucas can see where approvals are slowing output.
- Add a `reply engine` panel to Broadcast for growth: best performing replies, quote posts, and open engagement targets.

## Content Engine

- Surface `top hooks waiting` from review/content so the dashboard shows what can be published in the next hour.
- Add a `content pipeline pressure` band: drafted, in review, blocked on human, shipped.
- Add product launch tiles for `The $809 Mistake Guide`, `Advisor waitlist`, and any active Gumroad assets.
- Add a reusable visual pairing gallery so Lucas can preview post copy next to its card art inside the dashboard.

## Mobile / Share Mode Later

- Create a lightweight `Story Mobile` mode that collapses the timeline into stacked milestone cards and keeps a single giant CTA.
- Build a `share card` route for Story that opens cleanly from X, centers the milestone arc, and pushes people to follow Forge.
- Create a mobile-first Broadcast summary page: latest 3 posts, why Forge exists, and one follow button.
- Make the topbar shrink on scroll for mobile later so the cockpit identity stays visible without eating the viewport.

## Revenue / Trust Surfaces

- Show `cost per useful lesson` and `cost per shipped asset` so Forge’s efficiency advantage becomes obvious.
- Add a `what Forge is selling next` module once the first products are live.
- Add an `advisor waitlist / free questions` teaser once that lane is ready, but keep it honest and usage-transparent.
- Add an `operator receipts` panel: current model mix, daily burn, and why routing matters.
